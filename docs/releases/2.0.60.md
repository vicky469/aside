# Release Highlights

This release refines the index sidebar card workflow and makes deleted-note review safer.

## Highlights

- Adds read-only index sidebar tag filtering while keeping empty tag views empty instead of falling back to all comments.
- Makes index sidebar cards use the same selected-text preview treatment as individual note sidebars.
- Opens nested thread details by default and keeps deleted parent threads visible with their child entries.
- Restricts index cards to view/filter/resolve behavior by removing edit, add, delete, share, move, and retry actions.
- Adds per-card restore and permanent-delete controls in deleted mode, replacing the bulk permanent-delete toolbar action.
- Improves mobile index file clicks by revealing the sidebar when a file filter is selected.
- Tightens sidebar tab, toolbar icon, no-file empty-state, and thought-trail availability behavior.

## Notes

- The release artifact check covers `main.js`, `manifest.json`, and `styles.css` for source maps, embedded sources, local paths, and common secret patterns.
