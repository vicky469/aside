# Release Highlights

This release fixes SideNote2's open-note sync behavior when the note-backed comment block changes on disk outside the editor.

## Highlights

- Fixed external SideNote2 comment updates for already-open notes so managed-block changes can appear without closing and reopening the note.
- Added regression coverage for the external managed-block sync path and for the distinction between open editor content and stored file content.
- Refined the comment persistence controller to reuse the visible note body while refreshing comment state, derived links, editor decorations, and markdown previews.

## Notes

- This is not a CRDT merge layer. The auto-sync path only applies when the visible note body is unchanged and the difference is limited to the managed `<!-- SideNote2 comments -->` block.
