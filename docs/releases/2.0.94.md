# 2.0.94

## What's new

- Enforce repository-wide ESLint governance across maintained JavaScript, TypeScript, JSON, and JSONC files.
- Reject all inline ESLint configuration so local suppression comments cannot bypass the shared build policy.
- Keep test-specific exceptions centralized while production plugin code remains under strict type-aware linting.
- Test supported Node.js 22 and 24 LTS releases in CI.

## Notes

- This release changes build governance only; no user migration is required.
- Release artifacts remain limited to `main.js`, `manifest.json`, and `styles.css` and are checked for source maps, embedded source content, local paths, and obvious secret material before publication.
