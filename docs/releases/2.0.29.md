# Release Highlights

This release makes Aside's agent experience clearer during long Codex runs and reduces visual churn in the note sidebar.

## Highlights

- `@codex` requests now pack smarter default context from the current note, section or anchor, nearby headings, and recent thread transcript instead of relying on broad note dumps.
- Live agent replies now show real progress text from Codex when available, with a simplified footer that focuses on the spinner and progress instead of extra status labels.
- Cancelling an active agent run now stops the work while keeping the reply card in place, so the thread does not disappear mid-run.
- The note-level sidebar now reuses existing thread cards during refreshes, which reduces the flashing and full-card remounting that used to happen when comments were added or updated.

## Notes

- The `obsidian://aside-comment?...` protocol is still a navigation target for the UI; it is not yet expanded into agent prompt context automatically.
