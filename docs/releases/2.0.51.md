# Release Highlights

This release moves Aside canonical storage from inline hidden JSON blocks to per-note sidecar files, solving the large-note rewrite problem and making the plugin more reliable.

## Highlights

- **Per-note sidecar storage**: Side-note thread data now lives in `.obsidian/plugins/aside/sidenotes/by-note/<hash-prefix>/<full-hash>.json` instead of a trailing `<!-- Aside comments -->` block inside each markdown file.
- **Automatic migration**: On first startup after updating, the plugin scans all notes, migrates legacy inline blocks into sidecars, and strips the old managed blocks from source notes. No manual action required.
- **Zero source-note mutation after migration**: Once migrated, the plugin never rewrites the source note for comment operations. Only the small sidecar file is updated.
- **Support modal improvements**: Added an "Open data folder" button to the support inspector that opens the sidecar directory in your system file manager.

## Notes

- This release includes a one-time automatic migration. Vaults with many large notes will see a brief startup delay while blocks are converted.
- If a single note fails to migrate, the legacy block is left untouched and retried later.
- Repo-side helper scripts (`comment:create`, `comment:append`, etc.) now write to sidecars instead of inline blocks.
