# Release Highlights

This release hardens SideNote2 note storage so a markdown note must have exactly one managed `<!-- SideNote2 comments -->` block, and it refuses further writes when duplicate managed blocks are present.

## Highlights

- SideNote2 now rejects notes that contain more than one managed `<!-- SideNote2 comments -->` block instead of silently treating only the last block as managed.
- The CLI now reports a specific duplicate-block error so it is clear when a note must be repaired before creating, appending, updating, or resolving comments.
- Managed-block detection now uses a single shared analyzer for parsing, visibility, ranges, and write validation, which removes repeated rescans of the same note content.
- Added regression coverage for duplicate managed blocks in both storage helpers and the `comment:create` CLI path.

## Notes

- If a note already contains duplicate SideNote2 managed blocks, collapse it back to exactly one block before writing new comments.
