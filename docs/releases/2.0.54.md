# Release Highlights

This release fixes an editor jump after revealing a side note, adds a little more breathing room under the note toolbar, and makes GitHub release reruns safe when the same version already exists.

## Highlights

- Defer revealed-selection cleanup until after outside editor clicks settle so normal note clicks no longer jump back near the previously revealed anchor.
- Add extra bottom padding under the note toolbar in tags mode to keep the batch tag controls from feeling cramped against the content below.
- Make the release workflow idempotent when a GitHub release already exists for the pushed tag.
- Update release metadata and re-upload `main.js`, `manifest.json`, and `styles.css` on reruns instead of failing at `gh release create`.
- Keep the normal first-run release flow unchanged for brand-new version tags.

## Notes

- The jump fix affects local note clicks after a comment was previously revealed and you scrolled away from it.
- This release also includes maintenance for release automation, but it does not change the remote bridge feature set or note storage format.
