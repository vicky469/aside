# 2.0.87

## What's new

- **Experimental local publishing**: Aside can publish Markdown, HTML, and PDF files from the vault-relative `public/` folder to an existing Cloudflare Pages project using local Wrangler.
- **Pane-header publish actions**: Supported public files now expose publish, republish, unpublish, and open-published actions from the file pane header.
- **Public HTML preview support**: HTML files can open in Aside's in-vault HTML view and share comments with their paired Markdown source.

## Fixed

- Publishing state now stays consistent when standalone HTML/PDF files are renamed, deleted, paired with Markdown, or unpublished.
- Project name edits update generated `*.pages.dev` publishing URLs while preserving custom Publishing URLs.
- Publish artifact inspection blocks `.env*`, `.npmrc`, keys, certificates, source maps, log files, Obsidian private paths, and unsupported file types before deployment.

## Notes

- Publishing is experimental and requires Wrangler, Cloudflare login, and an existing Cloudflare Pages project.
- The vault's `public/` folder is the publish allowlist. A vault named `public` does not make root-level files publishable.
